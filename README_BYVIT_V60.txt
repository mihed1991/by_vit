Production MVP:
- Добавлен Node.js-сервер без внешних зависимостей.
- Данные товаров, страниц, отзывов, заказов и настроек админки сохраняются в data/store.json.
- Хранение вынесено в отдельный файловый адаптер: формат store.json не зависит от Railway и переносится на другой Node.js-хостинг без конвертации.
- Запись выполняется атомарно, перед изменениями создаются ограниченные резервные копии, доступные в разделе «Сервис» админки.
- Заказы и отзывы отправляются через API, а Telegram-уведомления отправляются с сервера.
- Bot Token, Chat ID и хэш пароля не отдаются публичной части сайта.
- Админка работает через серверную HttpOnly cookie-сессию.

Как запустить MVP:
1. Установи Node.js 20+.
2. В папке проекта выполни:
   npm start
3. Открой:
   http://localhost:3000

Проверка кода:
   npm run check

Переносимое хранение:
- BYVIT_STORAGE_DRIVER=file — текущий драйвер. Сервер использует единый адаптер, поэтому позже можно подключить PostgreSQL без изменения API и админки.
- BYVIT_DATA_DIR=/data/byvit — папка store.json. Укажите постоянный диск нового хостинга.
- BYVIT_BACKUP_DIR=/backups/byvit — отдельная папка резервных копий (необязательно).
- BYVIT_MAX_BACKUPS=12 — максимальное число серверных копий.
- BYVIT_STORAGE_PERSISTENT=true — отмечает подключённое хранилище как постоянное в /api/health и админке.
- BYVIT_BACKUP_TOKEN=случайная-длинная-строка — включает защищённый GET /api/backup для внешнего автоматического скачивания. Передавайте токен в заголовке Authorization: Bearer TOKEN.

Для переезда:
1. В разделе «Сервис» нажмите «Создать копию» и «Скачать с сервера».
2. На новом хостинге укажите BYVIT_DATA_DIR и запустите приложение.
3. Импортируйте скачанный JSON через админку. Товары, тексты, заказы, отзывы и аналитика сохранятся.

Важно:
- index.html напрямую всё ещё откроется как демо, но production MVP нужно запускать через npm start.
- data/store.json и data/backups создаются автоматически при первом запуске и не должны попадать в git.
- Для реального Telegram-бота нужен корректный Bot Token и один или несколько Chat ID в разделе Telegram админки.
- Для полноценного продакшена после MVP ещё нужны домен, HTTPS, резервные копии, нормальная БД, платежи, политика обработки данных и мониторинг.
