ByVit v60 — modern minimal rebuild

Что сделано:
- Полная пересборка проекта с чистым CSS вместо накопленных старых фиксов.
- Стиль приведён к референсу из byvit-store.zip: Inter, белый/молочный фон, строгий зелёный #2D5A27, тёмный текст, большие отступы, спокойные карточки.
- Страницы: index, catalog, product, cart, admin, brands, sale, wishlist, compare, delivery, stores.
- Каталог: поиск, рабочие фильтры по категориям/брендам, наличие, сортировка.
- Карточки: быстрый просмотр, избранное, сравнение, корзина, бейджи с редактируемым цветом, остатки, старая цена.
- Страница товара: фото, фасовки с изменением цены, вкусы, форма выпуска, страна, наличие, описание, состав, способ применения, отзывы, доставка, похожие товары.
- Корзина: количество, удаление, промокоды WELCOME/BYVIT10, способы доставки, оформление заказа.
- Telegram-заказ: в админке укажи Bot Token и Chat ID, после оформления заказ отправляется через Telegram Bot API без перехода пользователя в Telegram.
- Админка: вход по паролю, разделы товаров/главной/доставки/заказов/сервиса, редактирование описания, состава, способа применения, фасовок, формы выпуска, вкусов, цвета бейджа, hero-медиа, анимации, размеров заголовков/текстов блоков, адреса, доставки, Telegram-настроек, просмотр заказов, экспорт JSON, сброс демо.
- Обновление: разделы админки вынесены наверх и отсортированы по алфавиту, заголовки страниц уменьшены и приведены к единому стилю, поиск с главной отправляет запрос в каталог, для hero добавлен отдельный бегунок затемнения.
- Обновление 2: карточки каталога выровнены по цене, бейджи не накладываются, отзывы добавляются на модерацию и управляются из админки, добавлено массовое удаление товаров, редактируемые хедер/футер/заголовки страниц, настройки корзины и промокодов, аккуратное оформление заказа с автозаполнением адреса самовывоза.

Production MVP:
- Добавлен Node.js-сервер без внешних зависимостей.
- Данные товаров, страниц, отзывов, заказов и настроек админки сохраняются в data/store.json.
- Заказы и отзывы отправляются через API, а Telegram-уведомления отправляются с сервера.
- Bot Token, Chat ID и хэш пароля не отдаются публичной части сайта.
- Админка работает через серверную HttpOnly cookie-сессию.

Как запустить MVP:
1. Установи Node.js 20+.
2. В папке проекта выполни:
   npm start
3. Открой:
   http://localhost:3000

Проверка кода:
   npm run check

Важно:
- index.html напрямую всё ещё откроется как демо, но production MVP нужно запускать через npm start.
- data/store.json создаётся автоматически при первом запуске и не должен попадать в git.
- Для реального Telegram-бота нужен корректный Bot Token и один или несколько Chat ID в разделе Telegram админки.
- Для полноценного продакшена после MVP ещё нужны домен, HTTPS, резервные копии, нормальная БД, платежи, политика обработки данных и мониторинг.
